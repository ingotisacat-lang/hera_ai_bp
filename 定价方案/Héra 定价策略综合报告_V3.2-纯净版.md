## 执行摘要

### 核心建议

基于 Hera 当前状况（5M 日活职位、2-3 人团队、现金流未知），建议采用**"Data License 先行 → API-as-a-Service 规模化"**的两阶段策略。

### 三大支撑论点

| 论点 | 证据 | 含义 |
|------|------|------|
| **现金流友好** | Data License 启动资金仅 12 万美元，6 个月可上线产生收入 | 适合早期团队，降低资金压力 |
| **需求验证** | 通过 Data License 快速验证客户 WTP，降低 API 模式投入风险 | 先验证再投入，避免盲目开发 |
| **路径清晰** | Data License 客户可自然升级为 API 客户，获客成本摊薄 | 两阶段协同，非零和选择 |

### 关键假设

| 假设 | 验证方法 | 如不成立的备选方案 |
|------|----------|-------------------|
| **假设 1**: 6 个月内可签约 5-10 个 Data License 客户 | M2 末 review 签约进度 | 如<3 个客户，转向 Enterprise 模式试点 |
| **假设 2**: 12 个月内可完成 API 基础设施建设 | M6 末 review 技术进度 | 如延期>4 周，分阶段上线核心功能 |
| **假设 3**: 现金流能支撑 12 个月运营 | 每周现金流监控 | 如现金<6 个月，寻求 bridge financing |

### 主要风险

| 风险 | 概率 | 影响 | 风险等级 | 应对方案 |
|------|------|------|----------|----------|
| 客户 WTP 低于预期 | 中 (40%) | 高 | 🔴 高 | 调整价格至$10K/年或转向 Enterprise 模式 |

## 1. 报告目的

### 1.1 报告定位

本报告旨在为 Héra 项目的定价决策提供**决策工具**，而非仅信息参考。

**核心原则**：
- 无论公司最终选择哪种商业模式（Data License / API-as-a-Service / Enterprise Integration），本报告都能提供该模式下的最优定价模型
- 报告呈现**客观事实**和**多维分析**，由决策者基于自身情况判断
- 报告包含**明确建议**和**行动计划**，支持快速决策

### 1.2 报告要回答的问题

| 问题类型 | 具体问题 |
|----------|----------|
| **有哪些可行方案？** | Data License / API-as-a-Service / Enterprise Integration 三种模式的定义和区别 |
| **技术要求是什么？** | 每种模式对应的完整技术方案、技术难点、资源要求 |
| **成本结构是什么？** | 每种模式的固定成本、边际成本、持续运维成本 |
| **市场价格区间是什么？** | 同业竞争对手的价格水平和定价逻辑 |
| **财务模型是什么？** | 每种模式的盈亏平衡点、收入预测、利润空间 |
| **风险是什么？** | 每种模式的主要风险及应对方案 |
| **接下来做什么？** | 90 天详细行动计划和里程碑 |

### 1.3 报告不回答的问题

- 应该选哪种商业模式（由决策者基于现金流、团队能力、市场资源判断）
- 哪个方案"最好"（取决于公司目标和约束条件）
- 具体定价数字（报告提供价格区间和测算方法，具体数字需结合实际情况）

### 1.4 报告使用方法

```
决策者阅读路径（推荐）:
1. 阅读执行摘要（2 分钟） → 了解核心建议和关键决策
2. 阅读第 13 章决策者行动清单（5 分钟） → 明确接下来做什么
3. 阅读第 7 章建议（10 分钟） → 理解方案选择逻辑
4. 根据需要查阅技术/定价/竞争力分析（按需）
5. 参考附录了解理论模型和财务细节（按需）
```

## 4. 定价的理论依据

### 4.1 Simon-Kucher 价格三部曲理论

| 定价策略 | 定义 | 适用场景 | SKP 观点 |
|----------|------|----------|----------|
| **成本导向定价** | 基于生产成本 + 目标利润率 | 商品化程度高、差异化小的产品 | 仅作为底线参考 |
| **竞争导向定价** | 根据竞争对手价格水平设定 | 高度竞争市场、产品同质化严重 | 需结合竞争分析 |
| **价值导向定价** | 基于客户感知价值设定 | 差异化明显、创新性解决方案 | **最理想策略** |

**Hera 采用价值导向定价**，理由：
- 差异化优势明显（5M+ 职位覆盖，18 维度 Job AD，Company Info 独有）
- 目标客户支付能力强（大模型公司 B 轮+，估值$100M+）
- 成本结构不透明（数据获取成本难以精确计算）

## 10. 定价理论模型附录

### 10.1 Simon-Kucher 价格三部曲理论

| 策略 | 定义 | 计算公式 | SKP 观点 |
|------|------|----------|----------|
| **成本导向定价** | 基于生产成本 + 目标利润率 | 价格 = 成本 × (1 + 目标利润率) | 仅作为底线参考 |
| **竞争导向定价** | 根据竞争对手价格水平设定 | 价格 = 竞品价格 × (1 ± 差异化系数) | 需结合竞争分析 |
| **价值导向定价** | 基于客户感知价值设定 | 价格 = 客户感知价值 × 价值捕获率 | **最理想策略** |

### 10.2 价格瀑布模型（Price Waterfall）

```
标价 (List Price)
    ↓
├── 折扣 (Discounts)
    ↓
├── 促销 (Promotions)
    ↓
├── 返利 (Rebates)
    ↓
├── 物流成本 (Logistics Costs)
    ↓
├── 付款条件 (Payment Terms)
    ↓
实际净收入 (Net Realized Revenue)
```

**关键洞察**: 企业平均只能实现标价的 50-70%，30-50% 的收入在瀑布中流失。
